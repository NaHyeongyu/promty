# Codex Integration Validation

Goal: validate Promty Collector using a real Codex CLI `UserPromptSubmit` hook.

Do not use mock payloads for this validation path.

## Step 1: Configure Real Hook

The repo-local Codex hook lives at:

```text
.codex/hooks.json
```

Preferred setup:

```bash
python3 collector/src/cli.py install-hooks --tool codex-cli
```

Full local setup with login, hook installation, and uploader startup:

```bash
python3 collector/src/cli.py init \
  --app-url http://127.0.0.1:5173 \
  --api-url http://127.0.0.1:8011
```

Raw capture hook used for Step 2:

```bash
python3 "$(git rev-parse --show-toplevel)/collector/src/cli.py" capture-raw \
  --output "$(git rev-parse --show-toplevel)/docs/real-codex-payload.json"
```

The active hook for Step 4 and Step 5 is:

```bash
python3 "$(git rev-parse --show-toplevel)/collector/src/cli.py" capture --tool codex-cli
```

The active Stop hook for code-change capture is:

```bash
python3 "$(git rev-parse --show-toplevel)/collector/src/cli.py" capture-changes --tool codex-cli
```

Neither command uploads to the backend.

After adding or changing the hook, open `/hooks` in Codex and trust the hook if prompted.

Check the local setup status:

```bash
python3 collector/src/cli.py doctor
```

## Step 2: Capture Raw Payload

Submit a real prompt in Codex CLI from this repository.

Expected file:

```text
docs/real-codex-payload.json
```

Do not edit this file by hand.

Do not create it from a mock payload.

Captured example:

```text
docs/real-codex-payload.json
```

## Step 3: Compare With Promty Event Schema

Compare `docs/real-codex-payload.json` with `docs/event-spec-v1.md`.

Identify:

```text
missing fields
renamed fields
extra fields
```

Update only the Codex adapter if necessary:

```text
collector/src/adapters/codex/
collector/src/adapters/common.py
```

Do not modify the backend contract for Codex-specific differences.

Observed real Codex fields:

```text
cwd
hook_event_name
model
permission_mode
prompt
session_id
transcript_path
turn_id
```

Schema comparison:

```text
missing fields: none for PromptSubmitted
renamed fields: none
extra fields: none
```

Notes:

```text
turn_id is a Codex UUID, not a numeric sequence
sequence is assigned by Promty's local sequence store
session_id is stable and can be used directly as the Promty session_id
project_id is derived from the git root for cwd when no explicit project_id is provided
```

## Step 4: Convert Real Payload

After the raw payload is captured and reviewed, switch the hook command to:

```bash
python3 "$(git rev-parse --show-toplevel)/collector/src/cli.py" capture --tool codex-cli
```

Then submit another real Codex CLI prompt.

Expected flow:

```text
Real Hook
|
v
PromptSubmitted
|
v
JSONL Queue
```

No backend upload yet.

Default queue path:

```text
~/.promty/events/<project_id>/<session_id>/events.jsonl
```

Default sequence path:

```text
~/.promty/sequences.json
```

Default session index path:

```text
~/.promty/session-index.json
```

Default change baseline path:

```text
~/.promty/change-baselines.json
```

## Step 5: Multiple Prompt Validation

Run multiple real Codex CLI prompts.

Verify:

```text
sequence increments correctly
session_id remains stable
project detection works
project/session queue partitioning is correct
events.jsonl ordering is correct inside each session
```

## Success Criteria

A real Codex CLI prompt generates:

```text
Real Hook
|
v
Promty Event
|
v
~/.promty/events/<project_id>/<session_id>/events.jsonl
```

No mock data is used for the final validation.
